## Security Principles

1. **Single Source of Truth** - Only Service Hub validates passwords
2. **Shared Secret** - Both systems use same JWT_SECRET
3. **Immediate Revocation** - Database check on every request
4. **Role-Based Access** - Middleware enforces technician-only
5. **No Password Storage** - Technician Tracking never sees passwords
6. **No Token Generation** - Technician Tracking never creates JWTs
7. **Audit Trail** - All auth goes through Service Hub

## Data Flow Summary

| Step | System | Action |
|------|--------|--------|
| 1 | Frontend | User enters credentials |
| 2 | Tracking | Proxy to Service Hub |
| 3 | Service Hub | Validate password with bcrypt |
| 4 | Service Hub | Generate JWT with userId, role, name |
| 5 | Service Hub | Return token to Tracking |
| 6 | Tracking | Pass token to frontend |
| 7 | Frontend | Store token, use for requests |
| 8 | Tracking | Validate JWT on each request |
| 9 | Tracking | Check is_active in database |
| 10 | Tracking | Enforce role = technician |
| 11 | Tracking | Process request if authorized |
